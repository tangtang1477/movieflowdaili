# 代理商门户：佣金导向重构 + H5 适配 + 邮箱验证

## 目标

把系统从"额度管理工具"重新定位为"零成本赚佣金平台"。所有信息以"未来能赚多少钱"为核心呈现，前期阻力降到最低；同时补齐 H5/公众号体验、邮箱验证流程、佣金提现引导。

---

## 1. 移动端 / H5 公众号友好适配

整套页面（登录、注册、忘记密码、控制台）做响应式：

- **断点策略**：以 768px 为界，小于则进入 H5 模式
- **登录 / 注册 / 忘记密码**：AuthLayout 在 H5 下变为单列，左侧品牌区折叠为顶部 Logo + 一句 slogan，表单铺满；按钮与输入框增大触控区（min-height 44px）
- **控制台**：
  - 左侧 sidebar 在 H5 下隐藏，改为顶部 hamburger 抽屉（用 sheet 组件）
  - 顶栏精简为 Logo + 用户头像
  - 卡片网格 `grid-cols-3` / `grid-cols-2` 在 H5 下降为 `grid-cols-1`
  - 表格在 H5 下转为卡片列表（每条邀请码一张卡）
  - 弹窗在 H5 下变为底部抽屉样式，按钮全宽
- **公众号友好**：禁用横向滚动、字号最小 14px、点击区域 ≥ 44px、不依赖 hover（hover 状态降级为 active）、安全色保证微信内置浏览器深色模式可读

## 2. 重塑核心叙事：从"额度管控"到"佣金奖励"

### 注册 / 登录后的欢迎弹窗（替换现有"开始使用前"）

不再引导联系我们，而是用奖励文案吸引用户立即生成邀请码：

- 标题：**「欢迎加入，最高可赚 50% 佣金 🎉」**
- 描述：「您每邀请一位用户充值，最高可获得 **50%** 佣金奖励。现在就生成专属邀请码，开始赚取第一笔收益。」
- 主按钮：**「立即生成邀请码」**（直接打开生成邀请码弹窗，跳过任何前置设置）
- 次按钮：**「先看看控制台」**（关闭弹窗）
- 不显示任何"未设置返佣比例"的警告条

### 控制台总览（账户总览）改版

把当前以"额度"为核心的信息架构换成以"佣金收益"为核心：

- **顶部主卡片：累计佣金收益**
  - 大数字：`¥ 0.00`（当前可提现金额）
  - 副信息：累计已提现 / 待结算 / 本月新增
  - 主按钮：**「申请提现」**（点击触发提现引导弹窗，见 §3）
- **3 列卡片**：邀请人数 / 充值人数 / 累计佣金（替代现有 总额度/已使用/剩余）
- **奖励规则说明条**：「每邀请 1 人充值，最高可获得 50% 佣金」+ 折叠展开看分级规则
- **去掉**：可用额度预警（开关 + 阈值输入）整块删除
- **保留**：邀请码生成入口、待办事项、快捷设置

### 提现引导弹窗（§2 主按钮 / 提现入口触发）

这是「联系我们」唯一被前置的地方：

- 标题：「申请提现」
- 内容：「您当前可提现金额为 `¥XXX`。提现需联系我们的合作经理审核，预计 1–3 个工作日到账。」
- 按钮：**「联系合作经理」**（保留现有联系方式）/ 取消

## 3. 注册邮箱验证流程

注册流程改为两步：填写资料 → 邮箱验证 → 进入控制台。

- **第 1 步（注册表单）**：现有表单保留；提交时调用"发送验证码"逻辑（前端 mock 即可），按钮文案变为「发送验证邮件」
- **第 2 步（验证页 `/verify-email`）**：
  - 显示「验证邮件已发送至 `xxx@xxx.com`，请查收并输入 6 位验证码」
  - 6 位验证码输入框（用 `input-otp` 组件）
  - 「重新发送」按钮 + 60s 倒计时（复用登录页的倒计时模式）
  - 「修改邮箱」链接（返回注册第 1 步）
  - 验证码输入完整后自动提交，校验通过 → 跳转 `/console`
  - 三态完整：默认 / 输入中（自动校验时 loading）/ 错误（红色边框 + 提示）
- 注册页底部新增提示：「我们将向您的邮箱发送一封验证邮件」

## 4. 取消「可用额度预警」

完全移除：
- 总览卡片底部的开关和阈值输入
- 相关 state（`quotaAlertEnabled`、`quotaAlertThreshold`）
- 相关 props 透传

## 5. 「使用统计」页面：佣金导向重构

替换现有的"注册人数 / 使用积分"图表，改成更能说明赚钱的指标：

- **顶部 4 个数字卡片**：
  1. 通过邀请码注册：`N` 人
  2. 其中已充值：`N` 人（转化率 X%）
  3. 累计充值金额：`¥XXX`
  4. **预计可提现佣金：`¥XXX`**（核心指标）
- **结算时间线**（关键新增）：
  - 标题：「按结算日期展示可提现金额」
  - 表格：`结算日期 | 充值金额 | 佣金比例 | 可提现金额 | 状态`
  - 示例：`2025-05-31 | ¥1,200 | 40% | ¥480 | 待结算`
  - 文案说明：「结算日为每月最后一天，结算后 T+3 可申请提现」
- **保留**：6 个月趋势图（柱状）+ 邀请码状态分布饼图，但 Y 轴指标换为「佣金收益」

## 6. 「账户设置」改名为「账户信息」 + 联系我们按钮统一

- sidebar 项 `账户设置` → **`账户信息`**（icon 改为 👤）
- 在「账户总览」和「账户信息」中，所有显示为 `未设置` 的字段（返佣比例 / 结算日 / 等）右侧的标签 / 链接，统一改为一个「联系我们」按钮，样式参考现在橙色警告条上的：
  - `btn-outline border-amber-300 text-amber-700 hover:bg-amber-100 px-3 py-1 text-xs`
- 点击后打开「联系我们」弹窗（展示微信号 / 邮箱 / 二维码）
