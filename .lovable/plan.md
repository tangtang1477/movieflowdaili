
# 代理商门户 UX 重构

## 问题分析

当前页面信息层级不清晰，用户不知道"这是干嘛的"。按钮缺乏交互反馈（hover/active/disabled），登录缺少忘记密码功能。

---

## 1. 登录页优化

- 添加 **"忘记密码？"** 链接，点击切换到忘记密码表单（邮箱输入 + 发送重置链接按钮）
- 发送后显示"已发送"成功提示，可返回登录
- 新增 `/forgot-password` 路由页面，复用 AuthLayout
- 表单校验：邮箱为空时按钮禁用（disabled 态）

## 2. 全局按钮三态系统

在 `src/styles.css` 新增按钮相关 CSS 变量，统一所有按钮的三态样式：

| 状态 | 主按钮（赤陶渐变） | 轮廓按钮 | 文字链接 |
|------|---------------------|----------|----------|
| **默认** | 赤陶渐变背景 | 边框 + 透明背景 | 主色文字 |
| **Hover** | 亮度提升 + 轻微上移阴影 | 浅背景填充 | 下划线 |
| **Active/按下** | 亮度降低 + 缩放 0.98 | 深背景 | 颜色加深 |
| **Disabled** | 灰色背景 + 降低透明度 + 禁止光标 | 同上 | 同上 |

所有页面的按钮（登录、注册、控制台、弹窗）统一应用。

## 3. 控制台信息架构重构

当前控制台平铺所有信息，缺乏引导。重构为更清晰的分区：

**顶部导航栏**：Logo/品牌名 + 用户头像/姓名 + 退出登录（替代右上角孤立按钮）

**区域 A — 欢迎 + 状态概览**
- 欢迎语："你好，Vic"
- 账号状态标签（ACTIVE 绿色徽章）
- 返佣比例状态（未设置时显示黄色警告提示条 + "联系我们设置"按钮）

**区域 B — 邀请码额度（3 列卡片）**
- 总额度 / 已使用 / 剩余额度
- 卡片带图标，数字醒目

**区域 C — 快捷操作**
- "生成邀请码"按钮（主按钮样式，醒目）
- 简短说明文字："生成邀请码分享给客户，客户注册后自动关联到您的代理商账号"

**区域 D — 邀请记录表格**
- 表头：邀请码、可用次数、已使用、预存积分、创建时间、操作
- 操作列：复制链接、下载二维码
- 空状态时显示插图 + "还没有邀请码，点击上方按钮生成第一个"

**区域 E — 账号信息（可折叠/收起）**
- 合并"当前登录用户"和"代理商资料"为一个区域
- 默认折叠，点击展开查看详情

## 4. 弹窗交互优化

**返佣提示弹窗**（首次进入自动弹出）：
- 标题更明确："开始使用前"
- 说明文字更友好："您的返佣比例尚未设置。您可以先生成 0 预存额度的邀请码体验功能，如需提升额度请联系我们。"
- 按钮三态完整

**生成邀请码弹窗**：
- 输入框带数字步进器（+/- 按钮）
- 确认按钮在数量为 0 时禁用
- 底部额度卡片用颜色区分（预留用橙色，剩余用绿色）

## 5. 注册页优化

- 表单校验：必填项为空时提交按钮禁用
- 密码不一致时显示红色提示
- 提交按钮三态完整
