# 代理商门户功能重构

## 目标

让整个流程更清晰、更易用：注册/登录 → 返佣提示弹窗 → 生成邀请码 → 控制台主页

## 实现步骤

### 1. 优化登录/注册页面

- 调整字体大小、间距，使其更贴近截图样式
- 点击"登录"或"创建账号"按钮后，使用本地状态模拟登录，跳转到 `/console`

### 2. 创建控制台页面 (`src/routes/console.tsx`)

按照截图4的布局，包含以下区域：

- **顶部**：CONSOLE 标签 + "代理商控制台"标题 + 退出登录按钮
- **当前登录用户**：姓名、邮箱、代理商ID、状态
- **代理商资料**：企业名称、邀请码前缀、联系人、联系电话、联系邮箱、结算日、返佣比例
- **邀请码额度**：总额度、已预留、剩余额度（3列卡片）
- **邀请记录**：主邀请码、已注册客户数、邀请链接、复制/下载二维码按钮
- **全部邀请记录**：生成邀请码按钮 + 邀请码列表表格
- 以上功能按照更有逻辑的方式组织信息

### 3. 返佣比例待设置弹窗

- 首次进入控制台时自动弹出（截图2）
- 显示提示文字"您可以立刻生成0预存额度的邀请码，如需提升额度，请联系我们"
- 两个按钮："联系我们"（轮廓按钮）和"生成0预存邀请码"（实心赤陶按钮）
- 点击"关闭"可关闭弹窗

### 4. 生成邀请码弹窗

- 点击"生成0预存邀请码"或控制台中的"生成邀请码"按钮时弹出（截图3）
- 表单字段：生成数量（数字输入）、邀请码可用次数（数字输入）、注册预存积分（数字输入）
- 底部显示：本次预留额度、当前剩余额度（两个统计卡片）
- "取消"和"确认生成"按钮
- 点击确认后，生成模拟邀请码数据，更新控制台

### 5. 数据与状态管理

- 使用 React useState 管理所有数据（纯前端模拟，无后端）
- 注册表单数据传入控制台作为用户/代理商资料
- 邀请码使用随机生成的字符串

## 技术细节

- 所有弹窗使用 shadcn Dialog 组件
- 保持现有暖色调设计系统
- 新增路由：`/console`
- 退出登录按钮回到 `/login`
