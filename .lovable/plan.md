## 修改：生成邀请码弹窗补充预存积分支付入口

### 1. 提示语扩写（src/routes/console.tsx 第 302 行）

把现有提示扩成两行，增加引导预存的话术：

```
💡 客户使用邀请码注册并充值后，您将根据返佣比例获得对应佣金。
   预存积分越多，可生成的邀请码额度越充足，邀请效率与收益上限同步提升，建议预存以保障推广不断档。
```

容器仍用 `rounded-lg border border-border bg-muted/30 p-3 text-xs text-muted-foreground`，两段用 `<p>` 分隔，第二段加 `mt-1.5` 与轻微强调色。

### 2. 弹窗底部新增两个支付按钮

在「取消 / 确认生成」按钮组上方，新增一栏「预存积分」操作区：

```
┌─────────────────────────────────────────────┐
│ 预存积分                                     │
│ ┌──────────────┐  ┌──────────────┐          │
│ │  [微信icon]   │  │ [支付宝icon]  │          │
│ │   微信支付    │  │   支付宝支付  │          │
│ └──────────────┘  └──────────────┘          │
└─────────────────────────────────────────────┘
```

精简实现：

- 两个按钮等宽、`flex-1`，高度 44px（H5 触控）
- 仅显示一个品牌图标 + 极简文字（"微信" / "支付宝"），不写多余说明
- 微信用绿色品牌色 `#07C160`，支付宝用蓝色 `#1677FF`，使用 outline 样式（边框为品牌色，hover 浅底）
- 图标使用内联 SVG（项目已用 lucide，但 lucide 没有这两个品牌 logo，因此手写 SVG 路径），保持 `h-5 w-5`
- 点击后出现对应的弹窗二维码（用虚假的代替就行）让用户直接扫码支付预存积分；后端接入后再补真实支付链路（不在本次范围）

按钮组位置：紧贴提示语下方，再下面才是「取消 / 确认生成」。
